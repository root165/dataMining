Apriori · 频繁项集与剪枝

Frequent Itemset · Candidate Set · Pruning · Apriori Algorithm

📦 频繁项集 Frequent Itemset

满足最小支持度阈值 (minSup) 的项集
s(X) = σ(X) / N ≥ minSup
例：若 minSup = 0.4（40%），共有 10 笔交易，
{油条, 豆浆} 出现 4 次 → s = 4/10 = 0.4 ≥ 0.4 → 频繁
{油条, 包子} 出现 1 次 → s = 1/10 = 0.1 < 0.4 → 非频繁

📋 候选项集 Candidate Set

未经支持度检验的项集
Apriori 生成规则：由 Lk-1 自连接生成 Ck。
即：频繁 (k-1)-项集两两合并，生成候选 k-项集。
例：L₁ = {A, B, C} → C₂ = {AB, AC, BC}（待检验）

✂️ 剪枝 Pruning

提前删除不可能频繁的项集，减少计算量
非频繁项集的超集 → 一定非频繁 → 直接剪掉
Apriori 剪枝原理：若一个项集的某个子集不频繁，
则该项集本身必定不频繁，无需计算支持度。

🔄 Apriori vs FP-Growth 对比

Apriori
✅ 基于频繁项集性质剪枝
✅ 逐层广度搜索
❌ 可能产生海量候选集
❌ 多次扫描数据库
FP-Growth
✅ 用 FP 树压缩数据库
✅ 不产生候选项集
✅ 通常只需扫描 2 次
✅ 适合海量交易数据
两者都解决关联规则挖掘第一阶段的问题：频繁项集生成
