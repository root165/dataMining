## 一、考察重点

1. 理解数据准备地位：数据质量、表现形式和代表性会影响算法选择、参数选取和最终结果；高质量数据应完整、一致、易处理、具有代表性，并包含反映问题实质的特征。
2. 数据收集：对项目所需数据进行甄别、认定并汇聚；数据源包括政务数据、企业数据、咨询公司数据、网络信息等；获取方式需权衡经济性、时效性、有效性。
3. 抽样本质：统计学抽样主要因为获取全样本成本高；数据挖掘抽样主要因为处理全样本计算成本高；抽样应有代表性、保留原有特征和感兴趣内容。
4. 抽样方法：简单随机抽样分有放回与无放回；系统抽样按顺序等距抽样但可能受周期性影响；分层抽样适用于层内同质、层间异质；整群抽样适用于群内异质、群间同质。
5. 数据集成：将不同来源、格式、特点的数据在逻辑或物理上集中，难点包括异构性、分布与传输、自治性与松耦合、语义/实体识别冲突。
6. 集成技术：数据仓库偏物理存储，查询快但实时性较差；数据联邦偏逻辑虚拟视图，实时性好但现场计算受源库限制；中间件提供统一接口和逻辑视图，灵活但开发难度较大。
7. 异常值检测：3σ原则使用[μ-3σ, μ+3σ]识别正态分布下区间外异常；IQR法使用Q1、Q3和IQR；也可识别严重偏离回归关系的点。
8. 数据变换：归一化把数据映射到[0,1]；Z-score标准化使均值为0、标准差为1；离散化把连续数值映射为有限区间或类别；PCA用于降维但主成分含义通常更抽象。
9. 编码重点：哈希编码空间复杂度低、适合高基数属性，但存在哈希冲突；有序编码只适合有顺序的特征。

## 二、知识点分析

### 知识点一：数据准备的地位与高质量数据要求

- 数据准备是建模前最重要的基础环节。数据质量、表现形式和代表性，会影响数据挖掘算法选择、参数选取和最终结果。
- 高质量数据应满足：完整、一致；易于处理、适合数据挖掘算法；具有代表性；包含能够反映问题实质的特征数据。
- 考法提示：简答题常问“为什么数据预处理重要”，答案要落到提高数据质量、模型准确性、稳定性和可解释性。
- 

1. 提高数据质量 — 处理缺失值、噪声、异常值，避免"垃圾进垃圾出"
2. 提高模型准确性 — 消除冗余/无关特征、标准化量纲，让算法学得更准
3. 提高模型稳定性 — 降低方差，减少过拟合（如离散化、正则化预处理）
4. 提高可解释性 — 特征选择/降维让模型更简洁，容易理解哪些变量在起作用

### 知识点二：数据收集与数据抽样

- 数据收集：对项目所需数据进行甄别、认定并汇聚，可来自政务数据、企业数据、咨询公司数据、网络信息等。
- 统计学抽样目的：全样本获取成本高。数据挖掘抽样目的：全样本处理成本高，尤其是数据体量大、维度高、算法跑不动。
- 抽样原则：抽取的数据应有代表性，保留原有特征，并保留感兴趣的内容。

### 知识点三：四种经典抽样方法对比

- 简单随机抽样：完全随机，分有放回和无放回。有放回常用于理论模拟，无放回常用于实际应用。
- 系统抽样：按一定顺序排列并等距抽样。如果数据存在周期性模式，可能导致样本偏差。
- 分层抽样：先分层，每层都抽。适用于层内同质、层间异质的总体，能提高代表性和精度。
- 整群抽样：先分群，随机抽取若干群并调查群内个体。适用于群内异质、群间同质的总体，优点是降低成本。

根据教材第3章的结构特点，举实际场景：
	分层抽样（层内同质、层间异质）

- 客户满意度调查：按"省份"分层，每个省都抽样本。因为各省的消费习惯、经济水平差异大（层间异质），但省内用户相对同质（层内同质）。每层都抽样能保证全国代表性。
- 产品质量抽检：按"生产线"分层，每条线都抽。不同生产线可能有不同故障率，每条线都覆盖才能准确估计整体质量。
- 整群抽样（群内异质、群间同质）
- 学校体检抽查：以"班级"为群，随机抽几个班，抽到的班全员体检。因为各班学生构成（男女比例、成绩分布）差不多（群间同质），且每个班内部本身就包含各种学生（群内异质），抽几个班就能代表全校。
- 居民入户调查：以"小区"为群，随机抽几个小区，对小区内所有住户上门访问。同一个小区的住户内部有收入、年龄等差异（群内异质），而小区之间结构相似（群间同质），比全市随机抽人上门更省成本。

| 维度           | 分层抽样                   | 整群抽样                     |
| -------------- | -------------------------- | ---------------------------- |
| 先做什么       | 先分层                     | 先分群                       |
| 抽谁           | 每层都抽人                 | 抽若干个群                   |
| 群内是否全调查 | 否                         | 是                           |
| 目的           | 提高精度                   | 降低成本                     |
| 层/群结构要求  | 层内同质、层间异质         | 群内异质、群间同质           |
| 适用条件       | 层间异质性大、层内同质性强 | 群间差异小、群内各个体差异大 |

### 知识点四：数据集成的定义、难点与技术路线

- 基本概念：将不同来源、格式、特点性质的数据在逻辑上或物理上有机地集中，提供全面的数据共享。
- 四大难点（核心考点）：
  - 系统与数据异构：不同数据源在系统、模式（如关系型、文本等）、定义、语义上存在差异。
  - 分布性与传输性：数据源异地分布，对网络传输的准确性、实时性、安全性要求高。
  - 自治性与松耦合：各个数据源是独立的，可能会自行调整数据结构而不会通知集成系统，要求系统具备高鲁棒性。
  - 数据结构的不完整性与不关联性：存在数据缺失、主外键不统一等问题。
- | 维度     | 中间件（混合方式）                     | 数据联邦（逻辑虚拟）                     | 数据仓库（物理汇聚）       |
  | -------- | -------------------------------------- | ---------------------------------------- | -------------------------- |
  | 数据存储 | 集成服务器上整合，对外提供接口         | 不实际保存数据，虚拟化视图               | 物理存储，复制/聚集数据    |
  | 实时性   | 较好                                   | 极好（直接查询实时源）                   | 较差（受 ETL 周期限制）    |
  | 查询性能 | 中等                                   | 较低（现场计算，受限于源库）             | 查询极快（提前加工好）     |
  | 数据内容 | 原始数据+中间生成结果                  | 仅源数据内容                             | 可包含转换结果             |
  | 适用场景 | 系统多但不想动原有库，兼顾效率与实时性 | 数据源多且杂、实时性要求高、不能跨域传输 | 大规模历史趋势分析（OLAP） |
- 数据联邦(Data Federation）详解
  - 定义:基于数据查询操作，从不同数据源完成数据汇集，构成虚拟化数据库的方法。
  - 核心特点:不实际保存/复制/迁移数据；只保存数据来源及连接信息:提供虚拟化集成视图;屏蔽多数据源的物理位置、结构和保存方式。
  - 元数据：描述流程、信息和对象的数据，涉及技术属性、业务定义和操作特征（即“描述数据的数据”）。
  - 优点：应用方便、数据实时性好、开发快捷灵活。
  - 缺点：性能较低、系统可用性低（依赖源数据在线）、服务器负担大、数据内容受限。
  - 联邦数据库系统（FDBS）：半自治多数据源联合体；紧密耦合（统一访问，扩展困难）VS松散合（统一语言访问，需解决语义异构）。
- 中间件 (MiddleWare）详解
  - 定义：位于异构数据源系统（数据层）和应用程序（应用层）之间，提供统一数据模式和数据访问通用接口。
  - 核心功能：向下协调各数据源系统，向上为应用提供统一数据逻辑视图；隐藏底层模式和数据访问通用接口层数据细节
  - 关键问题：构造逻辑视图、解决数据异构性/完整性/语义冲突
  - 优势：灵活性极高、实时性好、适合处理逻辑复杂的连接；保护企业应用软件开发和维护利益
  - 局限：开发难度大（每个源需开发适配器）、查询性能受限、不适合超大规模数据物理整合

### 知识点五：数据清洗与异常值检测

- 数据清洗处理缺失值、噪声、异常值、重复数据和不一致数据。
- 3σ原则：正态分布或近似正态分布下，正常区间为[μ-3σ, μ+3σ]，区间外可判为异常值。
- 3σ	要求数据近似正态分布	差（均值和标准差受极端值影响大）
- IQR法：IQR=Q3-Q1，通常低于Q1-1.5IQR或高于Q3+1.5IQR的点可视为异常。
- 回归关系检查：若某点严重偏离变量间应有关系，可能是异常或录入错误。

IQR法：IQR=Q3-Q1，通常低于Q1-1.5IQR或高于Q3+1.5IQR的点可视为异常。

对，就是按位置（排序后的排名）划分的，不是按数值距离。
计算逻辑：

1. 将所有数据从小到大排序
2. 中位数（Q2）：把数据分成上下两半
3. Q1（下四分位数）：下半部分数据的中位数（即第 25% 位置的值）
4. Q3（上四分位数）：上半部分数据的中位数（即第 75% 位置的值）

特点：

- 不要求数据服从正态分布（比 3σ 法适用范围更广）
- 对极端值不敏感，使用中位数位置而非均值，稳健性好
- 常用于箱线图（Boxplot）中标识异常点——箱线图上下须的边界就是 Q1-1.5IQR 和 Q3+1.5IQR

### 知识点六：数据变换、离散化、编码与PCA

- 归一化：将数据映射到[0,1]区间，便于比较、加权或满足算法输入要求。
- Z-score标准化：处理后数据均值为0、标准差为1，常用于依赖距离或梯度的算法。
- 离散化：把连续数值映射到有限区间或类别标签，可减少属性值个数、削弱奇异值影响、形成概念分层。
- 哈希编码：空间复杂度低，适合高基数属性，但可能出现哈希冲突。
- PCA：用于降维和数据规约，但主成分含义通常比原始变量更模糊、更抽象。
  PCA 用于降维，但主成分含义通常比原始变量更模糊、更抽象，因为每个主成分是多个原始特征的线性组合，失去了原有的业务可解释性。

作用（降维）：

- 消除冗余特征，降低计算复杂度
- 缓解过拟合
- 便于可视化（降到 2D/3D）

## 属性变换(变量变换)

| 方法       | 适用场景                              | 核心特征                |
| ---------- | ------------------------------------- | ----------------------- |
| 归一化     | 图像处理、神经网络（需加快收敛）      | 将数据映射到 [0,1] 区间 |
| 标准化     | 聚类、SVM、逻辑回归（大多数算法首选） | 均值为 0，标准差为 1    |
| 非线性变换 | 处理偏态数据、非线性特征增强          | 改变数据分布形状        |

- Min-Max 归一化（线性变换）：

  - 公式： **$x^{\prime} = \frac{x - \min}{\max -\min}$** 或 **$x^{\prime} = \frac{\max -x}{\max - \min}$**
  - 特点: 将数据完全映射变换到 [0, 1] 区间。
  - 目的:便于进行比较、便于加权处理、满足某些算法（如神经网络等）的特定输入要求。
  - 

  拆解理解：

  - 分子 x - min：计算当前值离最小值的距离（偏移量）
  - 分母 max - min：整个数据的跨度（极差）
  - 比值：当前值在整体跨度中所处的位置比例

  > 本质上是线性缩放 —— 保持数据内部的相对距离不变，只是把坐标轴重新标定到 0,1。
  >
- Z-Score 标准化（Standardization）:

  - 公式： **$z = \frac{z - \mu}{\sigma}$=Z-0/1=Z**
    （其中 *μ* 为均值， *σ* 为标准差）
  - 特点: 处理后的数据符合均值为 0, 标准差为 1的分布。变体中可用中位数取代均值, 用绝对标准差取代标准差。
- 小数定标规范化：

  - 通过移动属性值的小数点位置进行规范化，公式为 **$x' = \frac{x}{10^k}$**

## 离散化（Discretization）

- 定义： 把连续的数字变成类别的过程，也叫分箱。在取值区间中指定 n-1 个分割点，将数据划分为 n 个区间，一个区间中的所有值映射到同一个分类值
- 为什么要离散化：

  1. 数据规约、维规约
  2. 一些算法要求离散属性数据
  3. 产生概念分层结构，可在不同抽象层进行挖掘
  4. 消除奇异值带来的影响
  5. 减少属性值个数，使结果知识表达更简洁、更易理解、更易使用
- 关键问题： 选择多少个分割点？分割点位置如何确定？

  - 分割点数目一般由用户根据业务经验或算法需求确定
  - 分割点位置可用非监督/监督方法确定

    教材第3章提到离散化的作用，结合现实场景举例：
    场景	原始连续值	离散化后	为什么这么做
    信用评分	收入：32000, 85000, 120000 等	低收入 / 中等收入 / 高收入	业务解释更直观；消除微小收入差异对模型的影响
    年龄分段	年龄：23, 47, 68 等	0-18, 19-35, 36-55, 56+	避免线性假设（年龄与风险不是线性关系），分箱后模型更稳健

## 三、复习步骤

1. 先画流程：收集-抽样-集成-清理-规约-变换，记住每步解决什么问题。
2. 抽样单独对比：简单随机、系统、分层、整群按“怎么抽、适用结构、优缺点”整理表格。
3. 集成单独对比：数据仓库、数据联邦、中间件按“是否实际保存数据、实时性、查询性能、适用场景”整理。
4. 清洗和变换做计算题：练3σ区间、IQR异常判别、Min-Max归一化、Z-score标准化、离散化定义与作用。

## 四、易错点

- 分层抽样和整群抽样容易反：分层是层内同质、层间异质；整群是群内异质、群间同质。
- 归一化和标准化不要混：Min-Max到[0,1]，Z-score到均值0、标准差1。
- 不要说PCA让主成分含义更清楚；材料强调主成分含义通常更模糊/抽象。

## 五、测试练习方向

- **抽样方法对比**：分层抽样最适合哪类总体？
- **归一化**：Min-Max归一化的结果通常映射到哪个区间？
- **异常值检测**：正态分布下3σ原则的正常数据区间为[μ-3σ, ____]。
- **编码方法**：哈希编码适合高基数属性，但可能出现哈希冲突。
- **抽样方法横向对比**：比较分层抽样与整群抽样的差异。


## 六、例题

### 1.数据预处理是数据准备的主要工作。请思考和阐述数据预处理在数据挖掘中所处的地位和所发挥的作用。 (必考重点)

地位：
数据预处理是数据准备的核心环节，也是数据挖掘全流程中最基础、最关键的步骤。它位于原始数据与模型算法之间，直接影响后续算法的选择、参数选取和最终挖掘结果的质量。实践中数据预处理通常占项目总时间的 *60%~80%*。
作用（从四个方面展开）：
作用	具体体现
提高数据质量	通过数据清洗处理缺失值、噪声、异常值、重复和不一致数据，确保"垃圾进、垃圾出"不发生，使数据满足完整性、一致性要求
提高模型准确性	通过特征选择/降维（PCA）消除冗余特征、通过归一化/标准化统一量纲消除特征尺度差异，让距离类算法（KNN、K-Means、SVM）和梯度类算法（神经网络、逻辑回归）学得更准
提高模型稳定性	通过离散化削弱极端值影响，通过特征规约降低维度减少过拟合风险，使模型对新数据的泛化能力更强、波动更小
提高可解释性	通过降维、离散化、特征选择减少属性个数，使结果知识表达更简洁，业务人员更容易理解"哪些变量在起作用、规律是什么"

### 2.说明为什么数据挖掘前必须进行异常值检测和清洗。

1. 异常值扭曲统计量 — 均值、标准差、相关系数等对异常值极为敏感，会直接影响后续算法（如 K-Means、线性回归）的参数估计
2. 误导模型学习方向 — 异常值可能被模型当作"规律"来学习，导致模型偏向异常模式，降低对正常数据的拟合和泛化能力
3. 干扰特征选择与降维 — PCA、归一化等预处理步骤同样受异常值干扰，Max-Min 归一化中异常值会拉伸极差，使大部分数据被压缩到狭窄区间
4. 降低模型稳定性和准确性 — 不消除异常值，模型在训练集上表现不稳定，测试集泛化能力下降
   核心一句话：异常值是噪声而非信号，不处理会导致"垃圾进、垃圾出"。

### 3.数据归约（Data Reduction）

 — 在尽可能保持数据完整性的前提下，减少数据量，降低后续挖掘的计算成本。
主要有三种方式：
方式	方法
维度归约	减少特征数量
数值归约	用更小的数据量表示同样的信息
数据压缩	无损或有损压缩
作用：

- 降低计算复杂度，加快算法运行
- 减少存储空间
- 缓解过拟合（维度降低后模型更简单）
- 使结果更简洁易理解
